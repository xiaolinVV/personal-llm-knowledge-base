# 资料笔记

`20-资料笔记/` 是资料消化层，用来把视频、文章、书、课程、会议、聊天记录、概念解释和工具观察转成自己的中文笔记。

这里不是收藏夹。每篇笔记至少要完成一件事：把来源材料转成自己的理解、问题清单、使用边界或工程判断。

## 与其他目录的边界

- 只保存来源入口，不写理解：放 `../10-来源索引/`。
- 单篇或小来源簇的资料消化：放 `20-资料笔记/`。
- 跨来源、稳定、可复用的主题知识：升级到 `../30-稳定知识/`。
- 围绕一个问题形成结构化判断：升级到 `../40-问题研究/`。
- 需要跑代码验证判断：升级到 `../50-实验验证/`。
- 产出 PPT、报告、文章或方案：放 `../70-输出成品/`，再把新判断回写到 `../30-稳定知识/` 或 `../40-问题研究/`。

## 命名建议

```text
YYYY-MM-DD-topic.md
```

视频、文章或课程笔记建议包含：

- 今天读了什么；
- 核心观点是什么；
- 哪些信息值得后续研究；
- 哪些地方不清楚；
- 这个能力适合什么场景；
- 这个能力不适合什么场景；
- 哪些结论尚未验证。

## 升级规则

不要把同一个主题拆得到处都是。新笔记先按主题归档；只有满足下面条件时再升级：

- 升级到 `30-稳定知识/`：已经跨来源整理，能作为长期主题知识复用。
- 升级到 `40-问题研究/`：已经围绕明确问题形成判断、证据和边界。
- 升级到 `50-实验验证/`：需要验证一个具体 API、SDK、工具链、失败模式或工程判断。

旧文件不强制补元数据。复查或升级时再使用 [note 模板](../60-方法库/templates/note.md)。
