# 真实场景和开源项目调研计划

学完官方技术栈后必须做这一阶段。否则只是“会调用 SDK”，不是会做 Agent 系统。

## 调研目标

找到真实系统里 Agent 的常见模式：

- 工具边界怎么设计
- 状态怎么管理
- 人工审批放在哪里
- 如何做 trace 和 eval
- 如何处理失败、重试、权限、成本
- 多 Agent 是否真的必要

## 真实应用场景

优先调研这些方向：

1. 客服与售后
2. 企业知识库问答
3. 工单流转
4. 代码修复与代码审查
5. 浏览器自动化
6. 数据分析与报表生成
7. 投研和资料研究
8. 销售运营工作流
9. 文档处理与合同审查

## 长期横向主题

除具体场景外，增加一个横向研究主题：

- AI 应用层经济性与泡沫识别：重点跟踪推理成本、DAU 陷阱、ToB 项目制、垂直数据壁垒、工作流控制点、硬件和端侧 AI。入口文档见 `40-问题研究/use-cases/2026-05-17-ai-application-layer-economics.md`。

## 开源项目筛选标准

不要只看 star。按这些标准筛：

- 是否有真实工具调用
- 是否有状态管理
- 是否有 trace 或日志
- 是否有 eval 或测试样本
- 是否处理权限和审批
- 是否能本地跑起来
- 是否有清晰模块边界
- 最近是否维护
