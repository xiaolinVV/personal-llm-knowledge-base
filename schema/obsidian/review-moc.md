# 复盘 MOC

复盘目标不是整理得更漂亮，而是减少未处理材料、升级成熟判断、暴露未验证事项。

## 每周复盘

- 清空或处理 [raw/inbox](../../raw/inbox/README.md)。
- 检查 [raw/sources](../../raw/sources/README.md) 中是否有值得转成笔记的来源。
- 检查 [wiki/notes](../../wiki/notes/README.md) 中是否有值得升级为研究或稳定主题的材料。
- 检查新输出是否需要回写到 [wiki/topics](../../wiki/topics/README.md) 或 [schema/methods](../methods/README.md)。

## 每月复盘

- 更新 [主题地图](../meta/topic-map.md)。
- 更新 [迁移记录](../meta/migration-log.md)，只记录真正改变仓库结构的决策。
- 从高价值笔记中提炼稳定知识。
- 删除、归档或降级长期停留在 `raw/inbox/` 的材料。

## 复盘视图

- [Knowledge Base](knowledge.base)：按属性查看待处理、研究、稳定知识和输出。
- 搜索 `未验证事项`：找出需要补来源、补实验或降级表述的内容。
- 搜索 `TODO` 或未完成 checkbox：找出半成品动作。
